Managing Pods

Table of Contents
  • 1. Overview
  • 2. Creating Pods
  • 3. Killing Pods
  • 4. List Running Pods

1. Overview

Pods are the computing units of {kube} and consist of one or more containers. A pod has its own IP address and storage resources. To scale an application, multiple pods with the same containers can be started. The pods are automatically distributed among the cluster nodes.

A very good introduction to starting an application in a pod is available at https://v1-11.docs.kubernetes.io/docs/tutorials/kubernetes-basics/deploy-interactive/.

For more information about pods, refer to https://v1-11.docs.kubernetes.io/docs/concepts/workloads/pods/pod-overview/.

2. Creating Pods

To create a pod, first define it in a YAML file. Usually, a pod has at least one container in it. The used containers must be available in a container registry, for example {suse} container image registry.

Example 1. Basic Pod with NGINX-Container
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  containers:
  - name: nginx
    image: nginx:1.7.9
    ports:
    - containerPort: 80

Then apply the YAML file with kubectl create -f [replaceable]YAML_FILE````.

3. Killing Pods

To kill a running pod, use kubectl delete pod [replaceable]POD_NAME`. The [replaceable]POD_NAME` refers to the name value of the metadata section. By default, the pod is granted a grace period of 30 seconds to shut down. After that, it will be killed.

4. List Running Pods

To list running pods, run kubectl get pods.
